# Task 8 — Capstone Project

**Duration:** 1 week (~8–10 hrs)

## Objective

Your portfolio centerpiece. Same freedom as Task 7, but polished to a
standard you'd show an employer: clear README, clean repo, a written
narrative, and a model/analysis you can explain and defend.

## Dataset

**You source it yourself.** Ideally pick a domain you're genuinely interested
in (sports, finance, health, climate, gaming, etc.) — motivation shows in the
quality of the questions you ask. Same minimum bar as Task 7 (1,000+ rows,
8+ columns, public/citable source), but aim for a dataset meaty enough to
support a real story.

You may extend/deepen your Task 7 project instead of starting fresh, if you
have a good reason to (state it in your README).

## Deliverables

In this folder:

1. `notebooks/` — your full pipeline, split into logical notebooks
   (e.g. `01_cleaning.ipynb`, `02_eda.ipynb`, `03_modeling.ipynb`) rather than
   one giant file.
2. `README.md` **specific to your capstone** (separate from this task
   description) containing:
   - Problem statement and why it matters
   - Dataset source and description
   - Approach summary
   - Key results (numbers + at least 2 charts, embedded as images)
   - Limitations and what you'd do next with more time
3. `src/` (optional but encouraged) — any reusable functions pulled out of
   notebooks into `.py` modules.
4. A model or analysis that answers a **clearly stated question** — not just
   "explore the dataset."

## Grading Criteria

- Would a stranger understand the project from the README alone, without
  opening the notebooks?
- Full correct pipeline: cleaning, EDA, modeling/analysis, evaluation
- Code is organized, not one 500-cell notebook
- Results are presented with appropriate charts and clear numbers
- Limitations are specific and thoughtful, not generic disclaimers

## After merging

This is the project to link from your GitHub profile / CV. Consider writing a
short blog post or LinkedIn post summarizing it — ask your mentor for
feedback on the write-up.
